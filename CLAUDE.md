## Overview

macdots is a symlink-based macOS dotfiles management system written in Zsh. All configurations are stored in `~/.local/share/macdots/configs/` and symlinked to their system locations (`~/.config/nvim`, `~/.config/zed`, etc.). This allows live editing and git version control.

## Architecture

### Core Components

- **`lib/core.sh`**: Shared library with logging, symlink management, backup functions, and platform detection. All installer scripts source this library.
- **`bin/macdots`**: Main CLI entry point. Routes commands (install, uninstall, update, doctor, backup, restore, status) to appropriate handlers.
- **`install.sh`**: Main orchestrator. Parses flags (--terminal, --desktop, --shell, --keyboard, --all) and runs corresponding installers.
- **`boot.sh`**: Bootstrap script for fresh installs. Clones repo to `~/.local/share/macdots` and runs installation.

### Installer Architecture

Installers follow a modular pattern:
- `install/homebrew/homebrew.sh` - Homebrew and packages
- `install/terminal.sh` - Terminal tools orchestrator (sources installers from `install/terminal/`)
- `install/desktop.sh` - Desktop apps orchestrator (sources installers from `install/desktop/`)
- `install/shell.sh` - Shell configuration (Oh-My-Zsh, plugins, .zshrc generation)
- `install/keyboard/karabiner.sh` - Keyboard customizations

Individual installers:
- `install/terminal/lazyvim.sh` - Neovim with LazyVim
- `install/terminal/k9s.sh` - Kubernetes CLI
- `install/terminal/krew.sh` - Kubectl plugin manager
- `install/terminal/aws-cli.sh` - AWS CLI
- `install/terminal/uv.sh` - Python package manager
- `install/desktop/zed.sh` - Zed editor
- `install/desktop/aerospace.sh` - Window manager
- `install/desktop/yazi.sh` - File manager

### Configuration Structure

Configs stored in `configs/` directory:
- `configs/zsh/` - Shell configuration
  - `aliases` - Command aliases (git, docker, kubectl, file system)
  - `shell` - PATH configuration
  - `init` - Tool initialization (mise, zoxide, starship, fzf)
  - `rc` - Main RC file that sources above files
- `configs/nvim/` - LazyVim Neovim configuration
- `configs/zed/` - Zed editor settings, keymaps, tasks, snippets
- `configs/aerospace/` - Window manager config
- `configs/yazi/` - File manager config
- `configs/k9s/` - Kubernetes CLI config
- `configs/karabiner/` - Keyboard customization config
- `configs/ghostty/` - Terminal emulator config

### State Management

- `.install_state` - Tracks installed components (homebrew, terminal, desktop, shell, keyboard) for idempotency
- `backups/` - Timestamped backups of replaced configs
- `install.log` - Installation log

### Shell Configuration Loading

The `.zshrc` file (generated by `install/shell.sh`) sources `~/.config/zsh/rc`, which in turn sources:
1. `shell` - PATH and environment setup
2. `aliases` - Command aliases
3. `init` - Tool initialization

Note: The shell config files reference `~/.local/share/macdots/defaults/zsh/` but these should actually be `~/.local/share/macdots/configs/zsh/`.

## Common Commands

### Installation
```bash
# Fresh install from bootstrap
curl -fsSL https://raw.githubusercontent.com/mrpbennett/macdots/main/boot.sh | zsh

# Install everything
./install.sh
# or
./bin/macdots install

# Install specific components
./install.sh --terminal
./install.sh --desktop
./install.sh --shell
./install.sh --keyboard

# Install with flags
./install.sh --dry-run          # Preview without executing
./install.sh --force            # Force reinstall
./install.sh --yes              # Skip confirmations
./install.sh --skip-backup      # Don't backup existing configs
```

### Management
```bash
./bin/macdots doctor            # Check system health and symlinks
./bin/macdots status            # Show installation status
./bin/macdots backup            # Create manual backup
./bin/macdots restore <timestamp>  # Restore from backup
./bin/macdots update            # Git pull and re-run installers
./bin/macdots uninstall         # Remove symlinks
```

### Testing Shell Scripts
```bash
# Validate shell script syntax
zsh -n script.sh
sh -n script.sh

# Test specific installer
source lib/core.sh && source install/shell.sh
```

## Development Patterns

### Creating New Installers

1. Place in appropriate directory (`install/terminal/`, `install/desktop/`, `install/keyboard/`)
2. Source `lib/core.sh` for utility functions
3. Use idempotency checks (`macdots_command_exists`, `macdots_is_installed`)
4. Use logging functions (`macdots_info`, `macdots_success`, `macdots_error`, `macdots_warn`)
5. Create symlinks with `macdots_symlink "${MACDOTS_ROOT}/configs/tool" "$HOME/.config/tool"`
6. Mark as installed with `macdots_mark_installed "component"`

### Shell Script Heredocs

When generating config files with heredocs in installers:
- Use single quotes for heredoc delimiter to prevent variable expansion: `'EOF'`
- Don't escape quotes unnecessarily within heredocs
- Example:
  ```bash
  cat << 'EOF' > file
  export EDITOR='vim'  # Correct
  export EDITOR=\'vim\' # Wrong - unnecessary escaping
  EOF
  ```

### Symlink Management

- All configs should be symlinked, not copied
- Check existing symlinks before creating: `macdots_symlink` handles this
- Backup existing files automatically unless `--skip-backup` is used
- Verify symlinks with: `macdots_verify_symlink "$target" "$expected_source"`

### Idempotency

- All installers should be idempotent (safe to run multiple times)
- Check if tools are already installed before installing
- Use `macdots_is_installed "component"` to check state
- Use `macdots_mark_installed "component"` to track installation

## OpenSpec Workflow

This repository uses OpenSpec for change management. Changes are tracked in `openspec/changes/` with artifacts (proposal, design, tasks, specs). Main specs are in `openspec/specs/`. Use the openspec skills (`/opsx:new`, `/opsx:apply`, `/opsx:verify`, `/opsx:archive`) for structured changes.

## Important Notes

- **Platform**: macOS only (Darwin). Scripts check with `macdots_is_macos`
- **Shell**: Zsh is the primary shell
- **Installation Location**: `~/.local/share/macdots/`
- **Backup Location**: `~/.local/share/macdots/backups/`
- **Log File**: `~/.local/share/macdots/install.log`
- **State File**: `~/.local/share/macdots/.install_state`
- All paths in scripts should use `$MACDOTS_ROOT` variable, not hardcoded paths
- The `bin/macdots` CLI should be added to PATH (done via `configs/zsh/shell`)
